# v0.1 概念驗證階段紀錄

此文件紀錄了在 v0.1 概念驗證階段中，為了實現「核心架構解耦設計」所進行的程式碼修改與命令列指令。

## 1. 程式碼修改與檔案結構

我們將原本單一的腳本拆分成了符合「三層架構」的模組：

### 1.1 新增：`data_ingestion.py`
- **用途 (資料擷取層)**：作為架構中的「插座」。此檔案實作了 `VirtualSensor` 類別，專門負責模擬產生隨機的溫濕度等環境數據，並將資料轉換成標準的 Python Dictionary (對應 JSON 格式)，使其可以輕鬆接入後續模組。

### 1.2 新增：`dlt_connector.py`
- **用途 (核心上鏈層)**：作為架構中的「引擎」。此檔案實作了 `IOTAConnector` 類別，負責把 IOTA SDK 的複雜邏輯（如節點連線設定、將 JSON 轉換成 Hex 十六進制字串、使用 `build_and_post_block` 發送資料等）封裝起來，提供一個簡單的 `send_data` 方法。

### 1.3 重構：`sensor_app.py`
- **用途 (驗證與展示層)**：作為開發階段 CLI 版的「戰情室」。此檔案被簡化成了程式的進入點 (Entry Point)，主要任務是匯入並初始化上述的 `VirtualSensor` 和 `IOTAConnector`，讀取資料後發送上鏈，並在終端機印出驗證結果 (Block ID)。

---

## 2. 執行過的命令列指令

### 2.1 執行解耦後的系統測試
```bash
.venv\Scripts\python.exe sensor_app.py
```
- **用途**：用於驗證 v0.1 的架構是否運作正常。因為專案有安裝 `iota_sdk` 在虛擬環境 (`.venv`) 中，所以明確指定呼叫虛擬環境下的 Python 直譯器來執行 `sensor_app.py`。此指令成功擷取了虛擬資料，並發送到 IOTA Shimmer 測試網路上。
